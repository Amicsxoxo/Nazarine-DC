# Nazerine-DC
Nazarine DC | Smart Energy Management System
Nazarine DC is a comprehensive, full-stack IoT monitoring and management platform. It is designed to provide real-time telemetry, historical data aggregation, and predictive insights for smart home and industrial energy grids.

The system bridges the gap between physical hardware (ESP32) and high-level data visualization, offering a unified design language and a robust data pipeline.

🚀 System Advantages
1. High-Fidelity Telemetry
Sub-Second Latency: Uses dedicated WebSockets to provide instantaneous feedback on power draw (kW), voltage, and current without the overhead of HTTP polling.

Hardware-in-the-Loop Design: Built to interface directly with ESP32 microcontrollers using standard JSON telemetry schemas.

2. Intelligent Data Aggregation
Integrated Accounting: The backend acts as a computational integrator, converting raw power (kW) into accumulated energy (kWh), real-time grid costs (₦), and carbon offset metrics (kg CO2).

Time-Series Grouping: Automatically aggregates thousands of raw data points into hourly and daily views using SQL-based grouping for cleaner analysis.

3. Professional UI/UX
Bento-Grid Architecture: A modern, clean design inspired by professional monitoring suites.

Unified Design System: Features consistent Tailwind CSS configurations across all pages, dark mode persistence, and glassmorphism effects.

Interactive Visuals: Dynamic, responsive charts powered by Chart.js that animate in real-time as data is received.

4. Scalability & AI Readiness
Modular Frontend: Pages like the AI Support Center and Analytics views are pre-wired into the global data bus.

Predictive Layer: Infrastructure is optimized for Pandas-ready data structures, enabling the future deployment of machine learning models for grid forecasting.

🛠️ Tech Stack
Frontend: HTML5, Tailwind CSS, JavaScript (ES6+), Chart.js.

Backend: Python 3.x, FastAPI, Uvicorn (ASGI), WebSockets.

Database: SQLite3 (Persistent Time-Series Storage).

💻 Local Setup & Execution
1. Prerequisites
Ensure you have Python 3.7+ installed on your system.

2. Install Dependencies
Open your terminal and install the required Python packages:

Bash
pip install fastapi uvicorn pydantic
3. Initialize the Database (Time Travel)
To avoid a blank dashboard, seed your database with 24 hours of historical telemetry:

Bash
python seed_db.py
Note: This generates a Nazarine_memory.db file in your directory.

4. Launch the Backend Server
Start the FastAPI server:

Bash
uvicorn main:app --reload
The server will be live at http://127.0.0.1:8000.

5. Open the Dashboard
Navigate to your project folder and open energy_dashboard.html directly in your browser.

Pro-Tip: Avoid using VS Code "Live Server" extensions. Because the database updates frequently, it may trigger an infinite refresh loop. Opening the file directly (file:///) is more stable.

6. Simulated Hardware Testing
If you don't have an ESP32 connected:

Open the Browser Console (F12 or Inspect).

Paste the JavaScript Simulator script provided in the documentation.

Hit Enter to begin beaming live telemetry to your server.
